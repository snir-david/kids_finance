## Technical Notes

### Key Decisions

1. **Emulator-first architecture** — all integration tests use local emulators by default
2. **Hybrid test strategy** — 40 tests runnable without infrastructure, 14 skip pending emulator
3. **In-memory fakes** — `InMemoryOfflineQueue`, `FakeSecureStorage` avoid heavy dependencies
4. **Custom `createdAt` in PendingOperation** — enables deterministic TTL boundary tests
5. **Extend concrete classes** — test fakes extend real implementations; avoids code generation

### Lessons Learned

- `FlutterSecureStorage` v10 uses `AppleOptions` (not `IOSOptions`/`MacOsOptions`)
- `FakeFirebaseFirestore` requires pre-seeded documents for `transaction.update()` calls
- `flutter_test` uses `bool?` for `skip` parameter (not `String`)
